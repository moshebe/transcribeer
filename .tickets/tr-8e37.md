Create TranscriptionService.swift — an @Observable @MainActor class that wraps WhisperKit for in-process transcription.

Responsibilities:
- Download/load WhisperKit model on first use (default: large-v3-turbo)
- Expose `transcribe(audioURL:language:) async throws -> [TranscriptSegment]`
- Report progress via published `progress: Double?` property
- Produce segments with (start, end, text) matching existing transcript format
- Store models in ~/.transcribeer/models/

## Design

```swift
struct TranscriptSegment {
    let start: Double  // seconds
    let end: Double
    let text: String
}

@Observable @MainActor
final class TranscriptionService {
    var progress: Double?
    var modelState: ModelState = .unloaded
    
    func loadModel(name: String) async throws
    func transcribe(audioURL: URL, language: String) async throws -> [TranscriptSegment]
}
```

## Acceptance Criteria

- Can transcribe a WAV file in-process without Python
- Progress updates during transcription
- Handles model download on first run with progress indication
- Supports language="auto" and explicit language codes

## Tests

- Unit test with short WAV fixture: segments non-empty, timestamps monotonic
- Test language=auto produces output
- Test model loading/unloading lifecycle
